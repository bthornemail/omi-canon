# State 16 — Symbolic Grammar: The Boundary

## Purpose

The `Omi-*` prefix establishes a symbolic protocol grammar independent of any
particular implementation.

The `Omi-` prefix is the boundary marker between categorical description and
canonical protocol grammar; outside that boundary OMI explains in Markdown, and
inside it OMI declares relations in OMI-Lisp dot notation.

Ordinary terms such as object, node, record, frame, message, document, and
receipt have accumulated different meanings across programming languages,
operating systems, networking, databases, distributed systems, mathematics, and
rendering tools.

OMI introduces a separate vocabulary whose meanings are fixed
constitutionally.

Implementations may project the grammar.

Implementations do not redefine the grammar.

```text
One grammar.
Many projections.
```

## Protocol Grammar Terms

### Omi-Notation

Omi-Notation is the symbolic representation of a relation.

It is syntax.

It is neither accepted nor rejected.

Examples:

```text
omi---imo
o---o
/---/
?---?
@---@
```

Omi-Notation declares structure.

### Omi-Citation

Omi-Citation is addressed notation.

It binds notation to deterministic scope.

A citation is addressable, but it is not yet validated.

Examples:

```text
|OMI---IMO>
o---o/---/?---?@---@
```

Omi-Citation addresses structure.

### Omi-Ring

Omi-Ring is the fully addressed notation witness.

It is the first stage where notation has become an addressed witness, but it is
not acceptance.

Recognition is not acceptance.

Citation is not acceptance.

Omi-Ring formation is not acceptance.

Projection is not acceptance.

Validation determines.

Omi-Attestation witnesses.

Accepted Omi-State may be recorded.

### Omi-Delta

Omi-Delta is the constitutional transformation primitive.

It names lawful deterministic transformation across OMI phases.

Omi-Delta owns the operation doctrine:

```text
XOR witnesses difference.
Rotation preserves orbit.
Unary composition constructs relation.
Shift only stages the next frame.
Validation determines state.
```

Omi-Delta is not merely an arithmetic operation.

It is the constitutional name for lawful deterministic transformation.

Examples include:

```text
Delta Law
Karnaugh reduction
Boolean minimization
combinator construction
unary composition
routing normalization
address normalization
```

Difference is witnessed.

Structure is preserved.

### Omi-Attestation

Omi-Attestation witnesses the result of validation.

It belongs to a citation.

It does not replace the notation.

Possible attested states include:

```text
candidate
validated
accepted
rejected
deprecated
merged
```

Omi-Attestation witnesses state; it is not the citation itself.

### Omi-Attribution

Attribution binds meaning to context.
