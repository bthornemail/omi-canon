2. One-word OMI rewrite instruction

A rewrite instruction is just an OMI 128-bit address frame used as an operation word. It selects a source rule/fact, derives a shadow canvas pointer, mutates existing browser nodes, and emits a receipt.

8. DevTools transaction bus

The runtime keeps a small in-page model of the SharedArrayBuffer bus. These operations round-trip cons pointers without needing a network transport.

Shared shape: SharedArrayBuffer(1024 * 16) = 16 KB, 128 instruction slots, 128 bytes per slot.

9. SpectrumDOM / PixelDOM output

The graph color field maps the OMI register split into browser-visible pixels: G(AA) = (V:RR, E:GG, I:BB, A:AA). Forward omi- nodes use direction:ltr; inverse imo- nodes use direction:rtl.

Source grounding

This page uses the field layouts of binary16, binary64, binary256, and block floating point as an analogy only. The OMI compiler model is a rewrite/projection system over OMI pointers, not a replacement for IEEE 754 arithmetic.
