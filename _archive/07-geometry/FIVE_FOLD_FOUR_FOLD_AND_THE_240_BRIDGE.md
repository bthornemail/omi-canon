# Five-Fold, Four-Fold, and the 240-State Bridge

## The Layer Distinction

OMI's factorial tower is not uniform. Each layer plays a distinct role. The most important hinge is between the five-fold layer (`5! = 120`) and the four-fold layer (`4! = 24`). This is where abstract packet identity becomes addressable projected state.

The lower factorial layers are structural:

```text
1! = 1     identity
2! = 2     complement / binary fold
3! = 6     line / S-P-O seed
4! = 24    selector / channel surface
5! = 120   packet root / simplex core
```

The higher layers are envelopes:

```text
6!  = 720     semantic sweep
7!  = 5040    Fano replay ring
8!  = 40320   8-segment physical frame
9!  = 362880  route/gate envelope
10! = 3.6M    declaration envelope
11! = 39.9M   witness/proof envelope
12! = 479M    clock/provider envelope
```

The five-fold and four-fold layers form the rule/fact transition core.

## Five-Fold Layer: `5! = 120`

The five-fold layer is the **packet root**. It represents the smallest complete packet universe before orientation, semantic sweep, Fano replay, or clock phases are applied.

Five equivalent characterizations:

```text
1. 5! packet core
2. 5-cell simplex (4D)
3. five semantic cells
4. rooted hidden factor inside 240
5. rule obligation core
```

### Five-Fold as 5-Cell Simplex

Geometrically, the five-fold layer maps to the **5-cell**, the 4D simplex. A 5-cell has five tetrahedral cells. OMI uses this as a model for a minimal complete semantic enclosure:

```text
five vertices / five cells / five roles
```

The packet is inside the 5-cell. Its meaning is stable because all five roles can be permuted without leaving the simplex boundary. `5! = 120` is the count of all role-orderings of the 5-cell packet core.

A five-fold packet model:

```text
1. subject seed
2. predicate seed
3. object seed
4. context/witness seed
5. closure/check seed
```

Thus a five-fold packet is a complete semantic unit:

```text
subject + predicate + object + context + closure
```

### Five-Fold Rules

A five-fold rule protects the rooted packet core. It constrains:

```text
semantic identity
packet closure
payload coherence
simplex role order
source/fold relation
```

A five-fold rule asks:

```text
Is the packet complete?
Is the packet coherent?
Does the packet preserve its root identity?
Can the packet survive orientation/complement?
Can the packet enter the 240 bridge lawfully?
```

## Four-Fold Layer: `4! = 24`

The four-fold layer is the **selector surface**. It is where packet identity becomes projectable.

Five equivalent characterizations:

```text
1. 4! selector space
2. 24-cell face layer
3. FS/GS/RS/US channel plane
4. 4-bit nibble surface
5. fact projection hinge
```

### Four-Fold as Selector Space

`4! = 24` means there are 24 permutations of four distinct positions. In OMI, those four positions are the basic channel faces:

```text
FS = File / Frame / Storage
GS = Group / Graph / Global
RS = Record / Relation / Routing
US = Unit / Text / Terminal
```

A four-fold fact is a fact that can be projected into one or more of these channels.

### Four-Fold as 24-Cell Face Layer

Geometrically, `4! = 24` maps to the 24-cell, a self-dual polychoron. The 24-cell is important in OMI because it gives a self-dual projection layer:

```text
inside / outside
rule / fact
source / shadow
valid / failure
```

`4! = 24` is the first layer where the rooted 5-fold packet can become visibly selectable.

### Four-Fold as Nibble Logic

A nibble has 4 bits and `2^4 = 16` possible values. The active nibble surface is `2^4 - 1 = 15`. This connects to the 240-state bridge:

```text
15 × 16 = 240
```

Four bits give the carrier. Fifteen active states give the nonzero surface. Together with a full 16-state carrier rail, they form the active byte surface.

### Four-Fold Facts

A four-fold fact declares that a packet has become visible through a selector/channel surface. It answers:

```text
Which channel is active?
Which selector is used?
Which projection is visible?
Which surface shadows the source rule?
Which failure behavior applies?
```

## The 240-State Bridge

The five-fold and four-fold layers meet at **240**. The identities:

```text
5! = 120
240 = 2 × 5!
240 = 16 × 15
240 = 16 × 16 - 16
6! = 3 × 240
7! = 7 × 3 × 240
```

Interpretation:

```text
5!    = packet root
×2    = orientation/complement
240   = oriented packet surface
×3    = S-P-O semantic sweep
6!    = semantic sweep
×7    = Fano selector
7!    = replay ring
```

The five-fold packet becomes a 240-state field only after adding orientation. The four-fold selector becomes a 240-state field by producing the active byte surface:

```text
15 active nibble values × 16 carrier values = 240
```

Both meet at the same bridge:

```text
five-fold side:  2 × 5! = 240
four-fold side:  15 × 16 = 240
```

This is the key result:

```text
5-fold explains why 240 is semantically rooted.
4-fold explains why 240 is byte-addressable.
```

## The Hidden Five

The factor `5` appears explicitly in `5! = 120 = 2³ × 3 × 5`. But above the five-fold root, it is encapsulated inside 240:

```text
240 = 2 × 5!
240 = 2 × 120
```

From `6!` upward, the factor 5 never appears explicitly in the decimal stepping or period structure. It is buried inside the 240 constant.

This resolves the apparent tension with the decimal carrier of `1/73`:

```text
1 / 73 = 0.01369863 01369863 ...
Period = 8
Repeating block = 01369863
```

The period block contains no digit `5`. This is not a contradiction. The factor `5` is not walked as a decimal digit. It is held as the hidden packet root.

Root vs step:

```text
5 is not a moving pointer.
5 is the rooted packet simplex.
```

The five-fold rule is not something the decimal carrier steps to. It is the condition that makes the climb meaningful.

Canonical hidden-root formula:

```text
5! is rooted.
240 carries it.
4! exposes it.
6! sweeps it.
7! replays it.
```

## Canonical 5040 Slot Decomposition

The 5040 replay ring is `7! = 5040`. Using the 240 bridge:

```text
5040 = 7 × 3 × 240
```

Canonical slot formula:

```text
slot5040 = fano7 × 720 + role3 × 240 + local240
```

Where:

```text
fano7    ∈ 0..6
role3    ∈ 0..2
local240 ∈ 0..239
```

The `local240` may be derived from either:

```text
five-fold packet + orientation
```

or:

```text
four-fold selector + carrier
```

A replay slot can be read as:

```text
Fano selector + S-P-O role + oriented packet/selector state
```
