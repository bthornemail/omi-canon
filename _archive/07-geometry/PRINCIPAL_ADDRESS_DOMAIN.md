# Principal Address Domain

The Omi-Ring Domain can be described by analogy to a Principal Ideal Domain.

In algebra, a Principal Ideal Domain is an integral domain where every ideal can be generated by a single element. OMI adapts that intuition into an address doctrine:

```text
Every valid OMI object should be generated from a bounded address element.
```

That generating element is not just a number. It is an enclosed relation.

```text
omi---imo
```

The Omi-Domain behaves like a principal address domain because every valid object must reduce to a generating enclosure:

```text
frame
truth row
receipt
address
projection
```

A valid object is not an arbitrary cloud of meanings. It is generated by a frame that can be validated and replayed.

```text
Every accepted object is generated by one bounded Omi-Ring address.
```

## Unique Factorization of Addressed Meaning

A Unique Factorization Domain says that elements factor uniquely into irreducible parts, up to units and order.

OMI adapts this as a doctrine of decomposable meaning.

An accepted OMI object should factor into canonical parts:

```text
Omicron boundary
frame segments
truth row
Delta path
receipt
projection
```

A portal may render the same object many ways:

```text
card
map pin
SVG node
QR code
JSON Canvas node
receipt row
timeline event
```

But those are not new meanings. They are projections of the same factored state.

```text
One accepted object may have many projections,
but its accepted structure factors through the same frame, truth row, Delta path, and receipt.
```

## Short Canon

```text
Every accepted object is generated by one bounded Omi-Ring address.
Projection may vary.
Accepted factors remain frame, truth row, Delta path, and receipt.
```
