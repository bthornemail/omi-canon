# State 15 — Classification Terminology: Roles and Categories

Status: Definitive canon doctrine
Layer: Term classification / document categories / role mapping

## Purpose

Classification Terminology establishes the canonical roles, document categories, and term classifications used throughout the OMI canon. It classifies terms, roles, and document categories. It does not redefine doctrine.

## Document Categories

Documents in the canon are classified by their role:

| Category | Role | Example |
|----------|------|---------|
| **State document** | Records a numbered phase in the canon journey | All 00–24 spine docs |
| **Dev-doc** | Source/reference material, not definitive doctrine | `_archive/` |
| **Truth gate** | Declarative rule file (.omi) | `RULES.omi`, `FACTS.omi` |
| **Proof** | Formal or mathematical verification | `constitutional-grid/axioms/` |
| **Grid cell** | Constitutional verification cell | `constitutional-grid/Q1-A1/` |

## Role Mapping

Each spine document occupies a role within the architecture:

| Spine Range | Role |
|-------------|------|
| 00–12 | Construction spine — what must exist |
| 13 | Master Architecture — map, not authority |
| 14–24 | Constitutional and conceptual closure — verification, classification, grammar, quotation, projection, carrier, reference |

## Authority Rule

Classification terminology classifies. It does not create authority.

Validation determines.
Omi-Attestation witnesses.
Accepted Omi-State may be recorded.
Projection only displays accepted relation state.
