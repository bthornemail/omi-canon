OMI Block Model · precision-page rewrite

From Floating-Point Pages to OMI Address Blocks

This page rewrites half precision, double precision, octuple precision, and block floating point as an analogy for the OMI block model. The source pages define real numerical formats; the OMI layer uses their field structure as a model for address, route, proof, clock, and shared-block semantics.

Core analogy

Floating-point page model

sign | exponent | significand

A number is interpreted by sign, scale, and payload precision.

→

OMI block model

polarity | route/scale | address payload

An OMI block is interpreted by complement polarity, envelope scale, and 128-bit address payload.

Annotation: We are not claiming OMI is a floating-point number. We are borrowing the field logic: a compact binary object becomes meaningful only after a sign/polarity bit, a scale/exponent field, and a payload/significand field are interpreted together.

1. Source precision pages as structural models

Block FP

Shared exponent

Block floating point assigns a group of significands to one common exponent. Microscaling formats usually group many small elements with a shared scale.

OMI rewrite: a block of OMI pointers can share one route, clock, or provider envelope.
binary16

Local header

Half precision uses 16 bits: 1 sign bit, 5 exponent bits, and 11 bits of significand precision, with 10 explicitly stored.

OMI rewrite: local Omicron cell: polarity, small scale window, compact payload/selector rail.
binary64

Clock bridge

Double precision uses 64 bits: 1 sign bit, 11 exponent bits, and 53 bits of significand precision, with 52 explicitly stored.

OMI rewrite: route/proof bridge: enough exponent width to model clock, envelope, and replay scale.
binary256

Macro envelope

Octuple precision uses 256 bits: 1 sign bit, 19 exponent bits, and 237 bits of significand precision, with 236 explicitly stored.

OMI rewrite: macro-provider envelope: global error-correction, witness, and high-range state blanket.

Derivation note: The floating-point pages show a repeated binary grammar: sign controls polarity, exponent controls scale, and significand carries payload precision. OMI rewrites that grammar into an address grammar: segment rails carry pointer structure, while derived envelopes supply route, declaration, proof, and clock scale.

3. The OMI block model

Physical block

omi-S0-S1-S2-S3-S4-S5-S6-S7/prefix

The physical carrier is still 128 bits: eight 16-bit segments. This is the fixed wire frame.

Derived envelope

pointer128
+ route9
+ declaration10
+ witness11
+ phase12

The higher factorial layers are not extra wire fields. They are deterministic interpretations wrapped around the same pointer.

4. Rewrite function

The rewrite function turns source precision terminology into OMI block terminology.

5. Three OMI precision pages

Half Page — Local Omicron

binary16
1 sign
5 exponent
11 precision

OMI model: local instruction header, compact selector, emoji/POS feature rail, small visual state.

Use for: local lexical routing, color preset selection, compact S-P-O payload hints.

Double Page — Clock Bridge

binary64
1 sign
11 exponent
53 precision

OMI model: bridge between local blocks and runtime clocks; stable enough for route, declaration, proof, and replay metadata.

Use for: HGV perceptron bridge, QEMU clock phase, WAN telemetry cadence, provider lane.

Octuple Page — Macro Envelope

binary256
1 sign
19 exponent
237 precision

OMI model: macro-envelope for large-range witness, ECC-like protection, cross-provider proof blanket.

Use for: global proof envelope, high-range semantic witness, large provider topology, archival receipt.

6. Block floating point becomes the OMI block provider model

BFP source idea

one exponent
shared by k significands

A group of values shares a scaling factor.

→

OMI provider idea

one envelope
shared by k OMI pointers

A group of pointers shares a route, declaration, witness, or clock phase.

Derivation note: This is the strongest source analogy. Block floating point proves that a block can preserve many local payloads while sharing one scaling context. OMI uses the same pattern: many 128-bit pointers can share one provider envelope, visual projection, telemetry cadence, or semantic route.

Final model

Floating-point page:
  sign + exponent + significand

OMI block page:
  complement polarity + envelope scale + address payload

Block-floating page:
  shared exponent + many local significands

OMI provider page:
  shared route/clock/witness envelope + many local OMI pointers

The purpose of this page is to make the analogy inspectable. The source formats give the field grammar; OMI rewrites that grammar into a protocol block model for pointers, rules, facts, proofs, clocks, and visible state.

Source notes

Based on the uploaded source pages for block floating point, half precision, double precision, and octuple precision. The OMI mapping is an analogy and design derivation, not a claim that OMI is numerically identical to IEEE floating-point arithmetic.
