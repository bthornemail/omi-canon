## SETCO Mental Boundary Mnemonic

SETCO may classify any idea, claim, object, concept, artifact, or relation by reducing it to three geometric roles:

```text
vertex / node
edge / link
face / graph
```

These correspond to three cognitive questions:

```text
Node:
  What is it?

Edge:
  How does it relate or move?

Face:
  What surface or context makes it readable?
```

The same structure may also be read through computational mode:

```text
state
phase
transform
```

And through ordinary grammar:

```text
noun
adjective
verb
```

## SETCO Classification Question

For any idea, ask:

```text
1. Is this a node?
   Is it an object, claim, fact, rule, proof, proposal, artifact, or address?

2. Is this an edge?
   Is it a transition, dependency, promotion, propagation, proof step, lowering, or transform?

3. Is this a face?
   Is it a context, classification plane, graph surface, projection, canvas, or interpretive frame?
```

Then ask:

```text
state:
  what remains stable?

phase:
  what orientation or context is visible?

transform:
  what movement or operation occurs?
```

Then ask:

```text
noun:
  what is named?

adjective:
  what qualifies or colors it?

verb:
  what acts, moves, relates, or changes?
```

## Canon Statement

SETCO uses geometry as a mental boundary and constraint mnemonic.

Any idea may first be classified as a node, edge, or face.

A node carries state.

An edge carries transform.

A face carries phase.

In ordinary language:

```text
node  = noun
edge  = verb
face  = adjective / context
```

This gives SETCO a universal classification handle before proof, implementation, projection, or interpretation are allowed to act.

The purpose is not to reduce every idea to a toy diagram.

The purpose is to prevent category collapse.

Before asking whether a concept is true, executable, canonical, or useful, SETCO asks where it belongs:

```text
Is it a node?
Is it an edge?
Is it a face?

Is it state?
Is it phase?
Is it transform?

Is it noun?
Is it adjective?
Is it verb?
```

Only after that classification may the idea enter the blackboard, canvas, proof registry, implementation rail, or OMI declaration surface.
