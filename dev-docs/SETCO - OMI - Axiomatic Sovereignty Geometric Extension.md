## The Pentagonal Subfolder Definitions

The five Axiomatic Sovereignty subfolders form the pentagonal base:

```text
LOGIC_PROOFS/
STRUCTURAL_TYPES/
PROCEDURAL_SEQUENCES/
CANONICAL_AXIOMS/
SOCIAL_CONTRACTS/
```

The pentagonal base is the usable artifact stack.

It relates to the collapse of higher-dimensional understanding into a form that can be held, inspected, translated, forked, and reused.

### 1. LOGIC_PROOFS/

```text
Shape role:
  point of formal truth

Artifact role:
  proofs, invariants, formal derivations

Question:
  what is true?
```

`LOGIC_PROOFS/` holds claims that should be checkable as formal invariants.

This folder should not become a runtime.

It should not become a narrative argument by itself.

It gives the model a proof-facing surface.

### 2. STRUCTURAL_TYPES/

```text
Shape role:
  shape of data

Artifact role:
  pure types, constructors, encodings, structural inventories

Question:
  what is shaped?
```

`STRUCTURAL_TYPES/` holds the data geometries that can be translated across languages.

This folder carries the invariant:

```text
many languages can read the same structural truth
no language owns it
```

Haskell may express a shape.

Rust may express a shape.

C may express a shape.

Coq may express a shape.

No language becomes the owner of the structure it helps reveal.

### 3. PROCEDURAL_SEQUENCES/

```text
Shape role:
  ordered transformation

Artifact role:
  deterministic sequences, transition rules, reproducible procedures

Question:
  how does it move?
```

`PROCEDURAL_SEQUENCES/` holds deterministic motion without making one execution surface final.

It should define inputs, outputs, preconditions, postconditions, and transition boundaries.

It should not confuse a sequence with a platform.

The sequence can be implemented in many runtimes.

No runtime owns the sequence.

### 4. CANONICAL_AXIOMS/

```text
Shape role:
  attested boundary

Artifact role:
  axioms, bounds, preserved claims, canonical constraints

Question:
  what must remain bounded?
```

`CANONICAL_AXIOMS/` holds axioms that should be attested rather than merely placed as placeholders.

Canonical does not mean centralized.

Canonical means the claim can be compared, preserved, translated, and carried as a reference.

This folder should not become a claim of ownership over another observer's application.

### 5. SOCIAL_CONTRACTS/

```text
Shape role:
  voluntary relation surface

Artifact role:
  agreements, contribution expectations, cooperation records

Question:
  what do people choose to uphold together?
```

`SOCIAL_CONTRACTS/` holds real agreements.

This folder should not contain empty symbolic contracts.

An agreement belongs here when someone can actually understand what is being upheld, by whom, for what purpose, and with what consequence.
