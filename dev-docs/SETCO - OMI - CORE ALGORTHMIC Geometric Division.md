## SETCO / OMI Geometric Division

The architecture is divided into three complementary geometric models:

```text
▲ TETRAHEDRON
omi-object-model
Pure Relation / Declarative Faces

■ CUBE
setco-framework-model
Transition / Classification / Canvas Workspace

● SPHERE
Omnicron Embeddable Binary Core
Invariant Algorithmic Deterministic Core
```

### Tetrahedron — Omi Object Model

The tetrahedron represents the pure relational object model.

It is the minimal bounded structure of relation-facing surfaces.

```text
omi-object-model
  omi-notation
  omi-lisp
  omi-port
  omi-* declarative faces
```

These are declaration and relation surfaces.

They do not become application behavior by themselves.

They provide the pure OMI-facing grammar through which relation may be named, addressed, declared, and carried.

Canonical reading:

```text
The tetrahedron gives OMI its pure relational faces.
```

---

### Cube — SETCO Framework Model

The cube represents transition, classification, and staged understanding.

```text
setco-framework-model
  blackboard
  canvas
  classification
  categorization
  canonization
  adjudication
  promotion
  propagation
  hypercube graph workspace
```

The cube does not replace OMI.

It classifies what external claims, proofs, implementations, proposals, and projections are allowed to do.

Canonical reading:

```text
The cube gives SETCO its transition and classification workspace.
```

---

### Sphere — Omnicron Embeddable Binary Core

The sphere represents invariant deterministic closure.

It encapsulates the embedded binary core:

```text
Omnicron
Omicron
Metatron
Tetragrammatron
```

In this reading, Omnicron is deterministic enough for the algorithmic core when it is treated as a headless embeddable resolver/framing core.

It may expose surfaces, but it does not decide application behavior.

```text
Omicron
  closes the null-byte ring

Tetragrammatron
  validates bounded closure

Metatron
  transposes without changing decision

Omnicron
  frames, resolves, and exposes projection-safe surfaces
```

Canonical reading:

```text
The sphere gives OMI its invariant deterministic binary core.
```

---

## Final Geometric Lock

```text
▲ Tetrahedron
  pure relation
  omi-object-model
  omi-notation / omi-lisp / omi-port / omi-* faces

■ Cube
  transition and classification
  setco-framework-model
  blackboard / canvas / hypercube workspace

● Sphere
  invariant deterministic core
  Omnicron embeddable binary resolver
  Omicron / Metatron / Tetragrammatron / Omnicron
```

The tetrahedron declares relation.

The cube classifies transition.

The sphere preserves deterministic closure.

Together they bridge:

```text
topological relation
  → discrete classification space
  → isotropic deterministic execution
```

No layer replaces another.

The Omi Object Model remains the pure relation model.

SETCO remains the transition and classification framework.

Omnicron remains the deterministic resolver-facing binary core when held inside the no-application, no-side-effect boundary.
